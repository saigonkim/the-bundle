# 📦 The Bundle - 사용 가이드 및 QA 테스트 계획서 (Final)

본 문서는 [The Bundle] 서비스의 원활한 운영과 검증을 위한 종합 가이드라인입니다.

---

## 1. 📂 사용자 가이드 (User Guide)

### 1-1. 서비스 개요
- **Concept**: 지루한 금융이 아닌, 아이템을 모으고 키우는 '성장형 자산 관리'
- **Target**: 20대 사회초년생 및 소액 투자자
- **Core Value**: 단순 수익률을 넘어 '기다림의 가치'를 보상(Patience Score)으로 환산

### 1-2. 워크플로우
1.  **시작하기**: 랜딩 페이지에서 [시작하기] 버튼 클릭 (로딩 상태 확인 가능)
2.  **구독 활성화**:
    - 대시보드 상단 또는 구독 페이지에서 [구독 시작하기] 클릭
    - 현재 테스트 버전은 **Mock 결제**로 즉시 'Active' 상태가 됩니다.
3.  **번들 탐색**:
    - 메인 대시보드에서 이번 달의 **추천 번들**과 **선정 이유** 확인
    - 각 ETF 종목을 클릭하여 '비유로 설명된' 쉽고 재미있는 리포트 열람
4.  **수익률 및 지표 트래킹**:
    - 상단 메뉴의 [수익률 트래킹] 진입
    - 나의 누적 수익률(ROI) 추이와 **기다림 단계(씨앗->새싹->나무->현인)** 확인

## 2. 🛡 관리자 가이드 (Admin Guide)

### 2-1. 관리자 대시보드 (`/admin`)
- **목적**: 매월 바뀌는 테마 번들을 AI를 통해 생성하고 게시합니다.
- **주요 기능**:
    - **번들 생성**: 10가지 고유 테마(AI 혁명, 반도체 패권 등) 중 하나를 선택하여 AI 초안 생성
    - **리포트 검토**: AI가 생성한 ETF 조합, 비유(Metaphor), 선정 이유(Rationale)를 확인합니다.
    - **게시(Publish)**: 초안이 마음에 들면 [발행하기] 클릭. 이때 기존 게시된 번들은 자동으로 '과거 기록(Archived)'으로 이동합니다.

### 2-2. 데이터 수집 자동화 (Cron)
- **Vercel Cron**: 평일 오후 3:30(장 마감 후)에 자동으로 최신 ETF 가격을 수집합니다.
- **동작 방식**: 
    1. KIS API 연동 -> ETF 현재가 및 NAV 수집
    2. 유저의 구독 시작가 대비 현재가 비교 -> `patience_logs`에 수익률 및 지수 자동 기록

## 4. 📈 데이터 검증 및 테스트 팁

### 4-1. 데이터 무결성 확인 방법
- **일간 업데이트 확인**: 매일 오후 4시 이후 `/dashboard/performance` 페이지의 '최근 기록' 리스트에 새로운 날짜의 행이 추가되었는지 확인하십시오.
- **수익률 계산법**: `(현재 NAV / 구독 시작 시점 NAV) - 1` 공식이 차트의 ROI와 일치하는지 샘플링하여 검토하십시오.

### 4-2. 시뮬레이션 활용 (테스트 환경 전용)
- **테스트 데이터 생성 버튼**: 대시보드 우측 하단의 [14일치 테스트 데이터 생성하기]를 클릭하면 즉시 그래프의 역동적인 변화를 확인할 수 있습니다.
- **DB 직접 수정**: Supabase SQL Editor에서 `update patience_logs set virtual_return_pct = 10.5 where ...` 명령어를 통해 급격한 변동 시 차트가 어떻게 반응하는지(예: Rose vs Emerald 색상 변화) 테스트 가능합니다.
